# 映射字典 (HashMap) (电商系统)

某电商系统最初使用一个 `List<Order>` 保存所有订单。每次根据订单编号查询或删除订单时，都需要从头到尾遍历列表并逐个比较订单编号。当订单数量增多时，这种方式效率较低。

请使用 `HashMap` 对订单存储方式进行重构，其中：
- Key 为订单编号（`String` 类型）；
- Value 为订单对象（`Order` 类型）。

要求完成以下功能：  
(1) 定义 `Order` 类，至少包含订单编号和商品名称；  
(2) 定义 `OrderManager` 类，使用 `HashMap` 保存订单；  
(3) 实现添加订单功能；  
(4) 实现根据订单编号查询订单功能；  
(5) 实现根据订单编号删除订单功能；  
(6) 编写 `Main` 测试类，演示添加、查询和删除操作，并体现 `HashMap` 在按订单编号查找时比遍历 `List` 更方便、更高效。

## 测试数据示例
请在 `Main` 类中使用以下测试数据进行演示：
- 添加商品：
  - `101`，`笔记本电脑`
  - `102`，`智能手机`
  - `103`，`蓝牙耳机`
- 查询商品：`102`
- 删除商品：`102`
- 删除后再次查询：`102`

**预期输出示例：**
```text
添加后商品数量：3
查询 id=102 的商品：智能手机
删除后再次查询 id=102：未找到
```
