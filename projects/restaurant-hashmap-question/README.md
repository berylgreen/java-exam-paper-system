# 映射字典 (HashMap) (餐饮系统)

餐饮系统原先使用 `List` 存储菜品信息，在根据菜品编号（ID）查找菜品时，需要遍历集合才能找到对应对象，效率较低。

请使用 `HashMap` 对该设计进行优化，其中：
- Key 为菜品编号 `String`
- Value 为菜品对象 `Dish`

要求完成以下功能：
1. 定义 `Dish` 类，至少包含菜品编号和菜品名称属性。
2. 定义一个管理类，使用 `HashMap<String, Dish>` 存储所有菜品。
3. 在管理类中实现以下方法：
   - 添加菜品信息
   - 根据 ID 查询菜品信息
   - 根据 ID 删除菜品信息
4. 编写 `Main` 测试类，演示添加、查询和删除操作，并体现使用 `HashMap` 后可通过键直接完成高效查找。
**测试数据示例：**
请在 `Main` 类中使用以下测试数据进行演示：
- 添加菜品：
  - `101`，`宫保鸡丁`
  - `102`，`鱼香肉丝`
  - `103`，`红烧肉`
- 查询菜品：`102`
- 删除菜品：`102`
- 删除后再次查询：`102`

**预期输出示例：**
```text
添加后菜品数量：3
查询 id=102 的菜品：鱼香肉丝
删除后再次查询 id=102：null
```
