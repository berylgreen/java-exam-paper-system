# 设计模式 (交通工具制造厂)\n\n目前的程序中，业务逻辑代码随处都在使用 `new Car()` 或 `new Truck()`，这不仅让客户端代码直接依赖具体产品，且当需要修改车辆初始化逻辑时，必须更改多处代码。

**任务要求**：
1. 定义接口 `Vehicle` 及其实现类 `Car` 和 `Truck`。
2. 创建一个静态简单工厂类 `VehicleFactory`，提供方法 `Vehicle createVehicle(String type)`。
3. 根据传入的类型（如 "car" 或 "truck"），在工厂内部封装创建对象的逻辑。
4. 修改主程序，移除对 `new` 关键字的直接使用，改为通过工厂类获取车辆对象并调用其 `drive()` 方法。
